SMS Inbox (no sign-in needed in the Shortcut)

This replaces the whole sign-in / Bearer token setup. The Shortcut just POSTs the raw SMS text — no authentication, no idToken, no headers to build correctly. The dashboard pulls new messages here and parses/categorizes them using the same logic as everywhere else.
You're running multiple keyword-based automations (credited/debited/received/sent etc.) so the same bank SMS often gets posted here more than once — that's fine, duplicates are auto-skipped when importing. This button just deletes the already-processed inbox documents afterwards, since they're no longer needed and would otherwise pile up indefinitely.
New Shortcut — one action, that's it:
"Get Contents of URL" → POST →
https://firestore.googleapis.com/v1/projects/—/databases/(default)/documents/sms_inbox
Headers: Content-Type: application/json (no Authorization header at all)
Body (JSON): {"fields":{"rawSms":{"stringValue":"[Shortcut Input]"},"processed":{"booleanValue":false}}}

Self-transfer detection

Transactions between your own accounts (e.g. moving money from one bank account to another) shouldn't count as real Income or Expense — they'll be bucketed into Transfer instead. This is detected by matching the name(s) below against who the SMS says the money is to/from, and by matching account numbers against your saved Bank Accounts.

Direct Firestore access (no billing required) — old approach, kept as fallback

This links an email/password sign-in to your same account (same data, same UID) so your Shortcut can authenticate directly against Firestore without needing a paid Cloud Function.
Values your Shortcut needs:
Project ID: —
Web API Key: —
Your UID: —

Step 1 — sign in (per run): POST to
https://identitytoolkit.googleapis.com/v1/accounts:signInWithPassword?key=YOUR_API_KEY
Body: {"email":"you@example.com","password":"yourpassword","returnSecureToken":true}
→ From the response, grab idToken as a variable.

Step 2 — write the transaction: POST to
https://firestore.googleapis.com/v1/projects/YOUR_PROJECT_ID/databases/(default)/documents/users/YOUR_UID/transactions
Header: Authorization: Bearer <idToken>
Body — you only need to send the raw SMS text and a timestamp, nothing else: {"fields":{"rawSms":{"stringValue":"full SMS text here"},"timestamp":{"integerValue":"1737000000000"}}}

The dashboard now extracts the amount, credit/debit direction, and category (Income/Transfer/Expense) automatically from rawSms — no need to parse anything in the Shortcut itself. This also handles messages that mention both "debited" and "credited" in the same text (common with account-transfer SMS) by picking whichever word sits closest to the amount.

Cloud Function key (only if you upgrade to Blaze)

Paste this into your Shortcut's "Get Contents of URL" action as header x-api-key. Endpoint: https://REGION-PROJECT.cloudfunctions.net/ingestTransaction · POST · JSON body: {"amount":1500,"type":"credit","rawText":"full SMS text"}
